## 概览

《暖湾》是一款轻魔幻治愈系孤岛生态经营游戏，机制范围围绕"种植生态→魔法干预→吸引动物→建立情感"的核心循环展开。系统设计的最高原则是**生态优先**——所有系统服务于生态成长，而非独立的数值膨胀。游戏包含37个系统，分布在 Core / Gameplay / Economy / Persistence / Narrative / Progression / UI / Audio / Meta 九个类别，覆盖从底层引擎基础到玩家感知的完整栈。技术方向为 Unreal Engine 5，C++ 核心系统 + Blueprint 游戏逻辑，目标平台 PC（Steam）+ 主机。

## 依赖关系图

依赖原则：从上到下设计与实现。上层系统是基础，下层系统是包装。

### 基础层（Foundation）—— 零依赖，最先设计

1. **输入系统** — 键盘/手柄/触屏映射，所有控制器依赖它
2. **游戏数据配置库** — 所有数值配置的数据驱动基础，所有 Gameplay 系统依赖它
3. **时间与天气系统** — 昼夜/季节/天气状态机，驱动材料采集窗口与生态节律

## 循环依赖检查

经分析，存在以下潜在循环需要关注：

| 涉及系统 | 风险描述 | 解决方案 |
|---------|---------|---------|
| **精灵魔法 ↔ 生态种植** | 魔法作用于植物，植物产出材料恢复魔力——存在数据依赖环 | 以"材料"作为解耦接口：魔法消耗材料/魔力，植物产出材料；两系统均读写资源系统，不直接调用彼此 |
| **动物情感纽带 ↔ 事件叙事** | 情感阶段触发事件，事件又推进情感阶段 | 事件叙事系统作为"触发器"，情感纽带系统作为"状态机"；触发器读取状态但不写回，由统一的"关系管理器"中介 |
| **岛屿生机度 ↔ 岛屿演变** | 生机度驱动演变，演变解锁新区域影响生机度计算维度 | 演变系统仅读取生机度阈值做状态切换，生机度系统不依赖演变系统的当前状态 |

## 高风险系统

| 系统 | 风险类型 | 风险描述 | 缓解策略 |
|------|---------|---------|---------|
| **动物行为AI** | 技术 + 设计 | 非战斗AI要表现出自然的觅食/社交/作息行为，同时不显得机械或可预期；UE5 行为树实现复杂度高 | 早期原型验证"动物看起来有生命感"的最低实现门槛；从里虹AI开始积累经验再扩展 |
| **植物生态联动系统** | 规模 | 50-80种植物的交互矩阵若全手动设计，工作量极大；组合爆炸风险 | 数据驱动架构强制要求：植物标签系统（而非一对一定义），联动规则作用于标签组合 |
| **存档系统** | 技术 | 需序列化完整世界状态——每棵植物的生长阶段、每位动物的位置与情感值、日志截图等；体量巨大 | 早期定义统一的世界状态快照接口；增量存档（只存变化量） |
| **自适应音乐系统** | 设计 | 音乐随生机度动态分层，过渡必须无缝且情感一致；对非线性音频经验要求高 | 与音频总监早期合作定义分层结构；参考《Sky：光遇》实现方案 |
| **有机新手引导** | 设计 | 无弹窗引导极难实现"恰好在玩家困惑前给出暗示"——需要精确的情境感知触发逻辑 | 扩展测试：记录玩家第一次种植发生的时间；超过N分钟无互动则触发里虹的额外暗示动作 |

## 最小原型系统集

> **核心验证问题**：不依赖任何奖励系统——单纯的「走近土地 → 催芽 → 破土动画 → 里虹嗅一嗅 → 生机度微动」30秒循环，本身是否令人愉悦？
>
> **原则**：原型 ≠ MVP。本集合仅需能运行出「感受」，不需要完整数值平衡、存档或 UI。

| 系统 | 原型角色 | 实现建议 |
|------|---------|---------|
| **生态种植系统** | 🔴 必建 | 实现种植/浇水/破土 3 个动作 + 生长动画；3 种植物硬编码即可 |
| **精灵魔法系统** | 🔴 必建 | 仅催芽 + 灵雨；视觉粒子效果是核心感受，施法数值全部硬编码 |
| **里虹伴侣AI** | 🔴 必建 | 只需「种植发生时跑来嗅/打滚」的单一即时反应，无完整行为树 |
| **魔力值系统** | 🟡 存根 | `const MAX_MANA = 100; SPELL_COST = 15;` 三行代码，无恢复逻辑 |
| **岛屿生机度系统** | 🟡 存根 | 单个整数计数器 + 地面颜色渐变（一个材质参数即可） |
| **土地感知系统** | 🟡 存根 | 固定返回「土地状态：可种植」的静态信息，无需动态感知 |
| **资源与材料系统** | 🟡 存根 | 硬编码 2 种散级材料，直接给玩家 5 个，无采集逻辑 |
| **游戏数据配置库** | 🟡 存根 | 全部 inline 常量，不需要真正的数据驱动架构 |
| 精灵角色控制器 | ⚪ 跳过 | 使用引擎默认角色移动组件 |
| 建造放置系统 | ⚪ 跳过 | 完全不需要 |
| 动物伙伴系统 | ⚪ 跳过 | 里虹存根已覆盖动物反馈 |
| 全部 UI 系统（5个）| ⚪ 跳过 | HUD / 背包 / 建造UI / 对话UI / 日志UI 全部跳过 |
| 全部 Narrative 系统（4个）| ⚪ 跳过 | 对话、事件、碎片、日志全部跳过 |
| 存档 / 音效 / 音乐 / 引导 | ⚪ 跳过 | 全部跳过 |

**范围结论**：37个系统 → 原型仅需 **3个必建 + 5个存根**，29个系统直接跳过

**验证时长目标**：1–3天等效工作量

**原型启动命令**：`/prototype 生态种植核心循环——催芽手感验证`
