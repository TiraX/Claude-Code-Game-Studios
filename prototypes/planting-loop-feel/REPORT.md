## Prototype Report: 生态种植核心循环——催芽手感验证

> **状态**: 待测试（代码完成，Blueprint 搭建中）
> **创建日期**: 2026-03-30
> **原型目录**: `prototypes/planting-loop-feel/`

---

### Hypothesis

我们假设：**不依赖任何奖励系统**（无背包、无 HUD、无成就、无对话），单纯的
「走近土地 → 种下种子 → 施展催芽 → 观察破土动画 → 里虹跑来嗅一嗅」
这条 30 秒循环本身，就足以让玩家想要重复这个动作。

核心赌注：破土的视觉粒子 + 里虹到达的时序感，是玩家驱动力的充分来源。

具体的可测假设：
- 玩家会在 **3 分钟内**不需要指引地自发种下第一粒种子
- 玩家会**自愿重复**种植循环 ≥ 3 次
- 玩家会**等待并观看**破土动画，而不是看一眼就离开
- 玩家会**注意到里虹的反应**，并对此有情绪回应

---

### Approach

**构建了什么：**
- `ProtoPlant`（C++）：种植、生长 tick、Seed→Sprout 阶段转换、OnSprouted 委托
- `ProtoSpellCaster` + `ProtoManaStub`（C++）：催芽 + 灵雨施法，纯 constexpr 魔力存根
- `ProtoLihong`（C++）：监听 OnSprouted → NoticeDelay 后跑向植物 → 嗅 → 打滚
- Blueprint 搭建说明（`BLUEPRINT_SETUP.md`）：粒子系统、动画蓝图、关卡布置
- 结构化测试脚本（`Test/PLAYTEST_SCRIPT.md`）：4 个测试区块，调参验证

**刻意跳过了什么：**
- 全部 UI（HUD、背包、对话、日志）
- 存档系统
- 动物伙伴系统（里虹存根覆盖动物反馈）
- 建造系统
- 全部叙事系统
- 音效（蓝图里加入，但不是原型瓶颈）
- 导航网格（里虹用直接移动代替）
- 错误处理、数据驱动架构

**关键可调参数（测试中重点观察）：**

| 参数 | 默认值 | 测试范围 | 验证的感受问题 |
|------|--------|---------|-------------|
| `NoticeDelay` | 0.6s | 0.0s / 0.6s / 1.5s | 里虹「感知到」的停顿是否增加生命感？ |
| `BASE_GROWTH_SEC` | 8s | 4s / 8s / 15s | 等待时长的满足感临界点在哪里？ |
| `CATALYZE_MULT` | 3× | 2× / 3× / 5× | 催芽的「爽感」是否足够？ |
| `SniffDuration` | 2.5s | 1.5s / 2.5s / 4s | 里虹嗅探时长 |

### Lessons Learned

> ⚠️ **待填写**：测试完成后总结

预期可能发现的洞察（测试前假设）：
- NoticeDelay 的最优值可能不是 0.6s——这个值影响「里虹有意识」vs「自动响应」的感知差距
- 催芽倍率需要达到某个临界值才能让玩家感受到「施法改变了结果」
- 破土动画的帧数比时长更重要——一个好的2帧抖动可能胜过流畅的20帧过渡
